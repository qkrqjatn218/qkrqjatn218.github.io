#무선 통신

무선 LAN

케이블 설치고민 없이 쉽개 네트워크 구축 가능 단말기의 이동이 자유로움

비싸고 느리다

IEEE 에서 무선 LAN 표준으로 208.11 ,208.11n, 208.11b, 208.11g 등 계속 지정하는데 현재 802.11n 규격이 자주 사용되고 있다

802.11n : 2.4GHZ 주파수와 5GHZ 주파수를 사용하며 최대 300Mbps 까지 전속속소를 제공함

802.11ac : 전송속도가 이론적으로 최대 6.93GB / 5GHZ 주파수 사용

802.11ax : Wi-Fi6 / 2.4GHZ 주파수와 5GHZ 주파수를 사용 / 5GHZ 주파수의 문제점인 약한 신호 개선

무선LAN 접근방식

CSMA/CA : CSMA/CD와 유사한 매체접근방식으로 패킷을 전송하기전 짧은 제어메세지를 먼저 전송하여 네트워크 사용여부를 파악하고 패킷을 전송한다

데이터 암호화 : 유선 케이블과 달리 무선 LAN 무선전파에 누구나 참여 할 수 있다는 문제가 있고 이에 대한 보안책으로 데이터 암호화 기술이 사용된다

WEP → WPA → WPA2 → WPA3(작년에 출시됨)

#근거리 통신

  • 이더넷(근거리 통신 프로토콜)

이더넷 헤더

>preamble - 이더넷 헤더에 포함되지 않음
         - 이더넷 신호가 전송됨을 알림

10101010 x 7 + 10101011

10101011 -> SFD (Start of Frame Delimeter)

>destination address 6바이트 -> 도착지 MAC 주소
<source Address 6바이트 -> 출발지 MAC 주소

>ethernet type : 상위 프로토콜 종류
ip : 0x0800
arp : 0x0806
..

>FCS : Freame Check Sequence : checksum 기능 4바이트 -> 오류체크하는 기능

이더넷 헤더의 길이 : 14바이트
이더넷 데이터의 길이 : 46-1500바이트

꼬리 4바이트를 제외한 이더넷 프레임의  최대 길이 : 1514바이트

와이어샤크 (wireshark) -> 패킷 캡쳐 프로그램 -> GUI를 통해 패킷 분석

패킷 리스트  : 캡쳐한 패킷 목록
패킷 디테일  : 선택한 패킷의 헤더 정보 정리
패킷 바이트  : 선택한 패킷의 실제 내용

tcp.port ==(포트 필터링) udp.port ==(포트 필터링)

tcp.dstport ==  tcp.srcport == 

||(or)   &&(and)

#포트 주소

동일한 ip를 가진 시스템 내에서 프로세스를 구분하기 위한 주소

0 - 65535 포트 사용

0 - 1023 : system port(잘알려진 서비스들이 사용함)

1024 - 49151 : user port(응용프로그램들이 사용함)

49152 - 65535 : dynamic port : 접속을 하는 용도로 사용

FTP : 21, 20   SSH : 22   TELNET : 23  SMTP : 25   DNS : 53 (udp)
TFTP : 69 (udp)  HTTP : 80   HTTPS : 443   pop3(메일관련 프로토콜) : 110
netbios : 137~139 (udp)  IMAP(메일관련 프로토콜) : 143   SMB : 445
원격 데스크톱 : 3389   SNMP : 161,162 (udp)  DHCP : 67 (udp)

#OSI 계층

application layer(7) : 어플리케이션이 실제 작용하여 서비스를 작동하는 계층

presentation layer(6) : 어플리케이션이 만들어내는 데이터를 표현하는 계층

session layer(5) : 송신자와 수신자의 연결상태를 관리하는 역할

transport layer(4) : 네트워크 전송과 관련하여 오류, 흐름제어 등의 기능을 통헤 신뢰성있는 정상적인 데이터 전송이 가능하게 함, 포트 번화를 통해 하나의 통신장치 안에서 각 어플리케이션을 구분을 가능하게 함

network layer(3) : 데이터를 목적지까지 가장 빠르게 갈 수 있는 경로를 찾음(라우팅)

data link layer(2) : 물리계층의 단순한 전기신호를 의미있는 데이터로 전환하여 인접한 네트워크 장비끼리 통신을 가능하게 하는 역할

physical layer(1) : 전기적, 기계적 규칙 등을 정의하여 네크워킈가 물리적으로 송수신이 가능하게 하는 계층

layer PDU Protocol 장비  
application layer data http, ftp, snmp,dns, dhcp L7  
presentation layer data      
session layer data      
transport layer segement TCP, UDP L4 switch port
network layer packet IP, IGMP, ICMP router, L3 switch IP
data link layer prame Ethernet switch MAC
physical layer bit   hub, repeater, cable, NIC  

#IP adress

인터넷에 연결된 모든 컴퓨터에는 고유의 주소가 부여

IP 주소는 8비트 크기의 필드 네 개를 모아서 구성한 32비트(4바이트) 논리 주소

일반 우편 주소를 시, 동, 번지 등으로 구분하는 것처럼 IP 주소도 네트워크 주소(Net ID)와 호스트 주소(Host ID)로 구분

  • A CLASS

    -2진법 = 00000000.00000000.00000000.00000000 ~

    ​ 01111111.11111111.11111111.11111111

    -10진법 = 0.0.0.0 ~ 127.255.255.255

  • B CLASS

    -2진법 = 10000000.00000000.00000000.00000000 ~

    ​ 10111111.11111111.11111111.11111111

    -10진법 = 128.0.0.0 ~191.255.255.255

  • C CLASS

    -2진법 = 11000000.00000000.00000000.00000000 ~

    ​ 11011111.11111111.11111111.111111111

    -10진법 = 192.0.0.0 ~233.255.255.255

  • D CLASS

    -2진법 = 11100000.00000000.00000000.00000000 ~

    ​ 11101111.11111111.11111111.111111111

    -10진법 = 224.0.0.0 ~ 239.255.255.255

  • E CLASS

    -2진법 = 11110000.00000000.00000000.00000000 ~

    ​ 11111111.11111111.11111111.11111111

    -10진법 = 240.0.0.0 ~ 255.255.255.255
