window 네트워크 기본 명령어

#네트워크 명령어

  1. ipconfig [네트워크 인터페이스 정보 확인]

    • 옵션

      형태 의미
      /all mac(물리적주소)주소 등
      /release ip 주소 해제하여 네트워크 연결을 끊음
      /renew ip 주소를 새롭게 갱신하여 네트워크 재연결

  2. netstat [사용중인 port 정보 확인]

  • 옵션

    netstat -an (전체 포트 정보 표시 및 포트 번호를 숫자로 표시)
     
netstat -ano (+PID정보)
     
netstat -anb (+실행프로세스)
-----------------------------------------------------------------------------
상태
LISTENING : 대기상태
ESTABLISHED : 연결됨
TIME_WAIT : 타임아웃
  1. nslookup [DNS 조회]

  • 방법

    nslookup
> [도메인네임]             (도메인네임에 해당하는 ip주소 출력)
  1. ping [상대방 호스트와 연결 상태를 확인하기 위해 사용, icmp 프로토콜 사용]

  • 방법

    ping [host ip주소]
-----------------------------------------------------------------------------
TTL(time to live) =128(윈도우)
TTL(time to live) =64(대체로 리눅스,안드로이드)
  1. tracert [해당 ip주소로 가는 경로 추적,icmp 프로토롤 사용]

  • 방법

    tracert [ip주소]
> 경로 출력
  1. arp [adress resolution protocol, ip주소를 mac 주소를 매칭]

  • 방법(근거리에서는 mac주소로 통신, 원거리에서는 ip주소로 통신)

    arp -a			(ip주소와 매칭되는 mac주소 캐쉬 확인)
arp -d			(캐쉬가 삭제됨)
  1. route print [라우팅 테이블 확인]

  • 방법(라우팅 테이블에는 최적의 경로에 대한 정보를 가지고 있음)

    route print
-----------------------------------------------------------------------------
네트워크 대상: 목표지점
인터페이스: 시작지점
메트릭: 숫자가 클수록 우선순위가 우위, 경로 지정시 필요

  1. hostname [호스트 이[름 확인]

  2. net user [사용자계정 확인]

#네트워크 형태

LAN(lan area network)

WAN(wide area network)

CAN(campus area network)

  • intranet(사설네트워크)

    -VPN(virtual private network) : 여러 곳에 퍼져있는 컴퓨터들을 잇는 사설 네트워크를 만들 때, 일일이 전선으로 연결하기는 돈이 많이 들고 물리적으로 보안에 취약하므로, 그 대신 인터넷 네트워크와 암호화 기술을 이용하여 통신 시스템을 구축하는 것을 말한다

#네트워크 접속장치

  1. LAN 카드(NIC카드) [외부 네트워크와 빠른 속도로 데이터를 송수신할 수 있게 컴퓨터 내에 설치하는 확장 카드를 말한다]
    • 각 LAN 카드에는 주민등록번호처럼 고유의 식별코드가 있다. 이렇게 LAN 카드에 할당된48비트 물리적 주소를 MAC 주소(Media Access Control Address)라고 한다
    • MAC 주소중 OUI(앞의 24비트)는 LAN 칩셋 제조사를 의미하는 고유 코드고, 뒤의 24비트는 제품의 일련번호와 같이 부여되는 것이므로 전 세계에서 유일한 내 컴퓨터만의 LAN 카드 고유 번호가 된다
  2. 허브(HUB) [여러 대의 컴퓨터를 손쉽게 연결하는 장치]
    • 한 포트에서 수신된 신호들은 허브의 다른 모든 포트로 즉시 재전송되며, 모든 입력과 출력은 서로 연결되어 있어 여러 개의 노드가 똑같은 중계기를 공유할 수 있도록 해준다
    • 단순히 전기적 신호를 복제해 여러 곳으로 전송하기 때문에 데이터 속도를 느리게 하고, 보안에도 취약함, 데어터 송수신이 동시에 안됌, 현재는 사용하지 않음
  3. 스위치(Switch) [컴퓨터에 할당되는 대역폭을 극대화시켜주는 장치]
    • 허브와 동일한 방식으로 작동하지만 수신하는 데이터의 의도한 대상을 식별할 수 있으므로 데이터를 수신하기로 되어 있는 컴퓨터에만 해당 데이터를 보낸다. 또한 스위치는 데이터를 동시에 송수신할 수 있으므로 허브보다 빠른 속도로 데이터를 보낼 수 있다
  4. 브릿지(Bridge) [두 개 이상의 근거리 통신망을 연결하여 하나의 네트워크로 만들어 주는 장치]
    • 근거리 통신망에서 하나의 장치가 데이터를 송신할 때 다른 장치도 데이터를 송신하면 충돌이 발생한다. 이처럼 네트워크에 노드 수가 늘어나면 충돌이 발생할 확률이 높고 통신속도도 떨어지는데, 브리지를 이용하면 문제를 해결할 수 있다
  5. 게이트웨이(게이트웨이 주소와 다름) [종류가 다른 두 개 이상의 네트워크를 상호 접속하여 정보를 주고받을 수 있는 장치다]
    • 브리지와 달리 서로 다른 프로토콜 통신망 간에도 프로토콜을 변환하여 정보를 주고받을 수 있다
  6. 중계기(Repeater)
    • 접속 시스템의 수를 증가시키거나 네트워크 전송 거리를 연장하려고 사용하는 장치로, 네트워크에서 신호를 수신하여 증폭한 후 다음 구간으로 재전송하는 역할을 한다
    • 장거리로 전송하면 신호가 약해지거나 감쇠되는데, 중계기는 노드 사이의 케이블에서 신호를 증폭시켜 이문제를 해결한다. 중계기는 신호를 증폭시키기만 할 뿐 데이터 내용은 변경하지 않으며, 근거리 통신망을 구성하는 세그먼트들을 확장하거나 서로 연결하는 데 주로 사용한다
  7. 라우터(Router) [서로 구조가 다른 망을 연결할 수 있어 근거리 통신망(LAN)과 대도시 통신망(MAN), 광대역 통신망(WAN)을 연결하는 데 사용]
    • IP 주소를 바탕으로 데이터가 수신지까지 갈수 있는 경로를 검사하여 효율적인 경로를 선택하는 라우팅 기능도 수행한다
    • 라우팅은 라우터에 있는 라우팅 테이블이 결정하는데, 라우팅테이블은 인터넷상에서 수신지의 주소를 토대로 경로상의 다음 주소를 결정한다
    • 라우터를 사용하면 서로 다른 네트워크(홈 네트워크와 인터넷) 간에 데이터를 전송할 수 있다. 라우터에는 이더넷 케이블을 사용한 유선 라우터와 무선 라우터가 있다

트리형(Tree)

#네트워크 전송매체

유선 전송매체(동축케이블, 꼬임선, 광섬유케이블)

무선 전송매체(라디오파, 지상 마이크로파, 위성 마이크로파)

#통신방식

다른컴퓨터에 데이터 전송 서비스를 제공하는 컴퓨터를 ‘서버’라 하고, 서버에서 보내주는 데이터 서비스를 수신하는 컴퓨터를 ‘클라이언트’라고 한다

  • 유니캐스트 1:1 통신 (서버와 클라이언트간에)
  • 멀티캐스트 1:다수 통신 224.0.0.0 ~ 239.255.255.255
  • 브로드캐스트 1:모두 통신 FF:FF:FF:FF:FF:FF

#근거리통신

  1. csma/cd(carrier sense multiple access with collision detection)
    • CSMA/CD(반송파 감지 다중 접근/충돌 검출) 방식은 버스형 통신망으로 알려진 이더넷에 주로 사용한다
    • CSMA/CD 방식은 데이터를 송신하기 전에 반송파 여부를 감지하는데,반송파가 감지되면 다른 컴퓨터(스테이션)에서 데이터 송신 중임을 판단하여 데이터를 전송하지 않는다
    • 하지만 반송파가 감지되지 않으면 컴퓨터가 전송매체를 사용하지 않는 것으로 판단하여 데이터 를 전송한다
